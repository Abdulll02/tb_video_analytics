# Telegram-бот для аналитики видео (Text-to-SQL)

Бот предназначен для получения аналитической статистики по видео на основе задач на естественном языке.
Сервис преобразует текстовый вопрос пользователя в SQL-запрос к базе данных PostgreSQL с использованием LLM, выполняет его и возвращает числовой ответ.

Проект выполнен в рамках тестового задания.

##  Как запустить проект

### Предварительные требования
- Установленный [Docker](https://www.docker.com/) и [Docker Compose](https://docs.docker.com/compose/).
- Файл с данными `data.json`. Вы можете скачать файл [здесь](https://drive.google.com/file/d/1BZOYxhDmMGJrSbPdcQgQjh0HRzN1YZt5/view?usp=sharing)

### Шаг 1: Подготовка данных
1. Создайте папку `data` в корне проекта (если её нет).
2. Положите скачанный файл `json` в эту папку и переименуйте его в `data.json`.
   Итоговый путь: `./data/data.json`.

### Шаг 2: Настройка окружения
Создайте файл `.env` в корне проекта и заполните его по шаблону:

```ini
# Telegram & LLM
BOT_TOKEN=ваш_токен_от_BotFather
OPENROUTER_API_KEY=ваш_ключ_от_OpenRouter
# Используемая модель (в нашем случае бесплатная от openrouter)
OPENROUTER_MODEL=arcee-ai/trinity-large-preview:free

# Настройки БД (оставьте как есть для Docker)
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
POSTGRES_DB=analytics_db
POSTGRES_HOST=db
POSTGRES_PORT=5432

# Путь к данным внутри контейнера
JSON_DATA_PATH=/app/data/data.json
```

### Шаг 3: Запуск
Выполните команду в терминале из корня проекта:

```bash
docker-compose up --build
```

**Важно:** При первом запуске автоматически сработает скрипт `load_data.py`. Загрузка файла (~500k записей) может занять 1-2 минуты. 
Дождитесь сообщения в логах: `Data loading finished successfully`. После этого бот готов отвечать на сообщения.

## 🏗 Архитектура и реализация

Весь стек реализован асинхронно (`asyncio`, `aiogram`, `asyncpg`) для обеспечения отзывчивости и высокой производительности.

### 1. База данных (PostgreSQL)
Данные из JSON нормализованы в две таблицы (см. `init.sql`):
- **`videos`**: Хранит итоговую статистику по каждому видео.
- **`video_snapshots`**: Хранит историю изменений (таймсерию) и почасовые дельты (приросты).
  
*Оптимизация:* Созданы индексы (`BTREE`) по полям дат (`created_at`, `video_created_at`) и внешним ключам для ускорения фильтрации и агрегации.

### 2. Загрузка данных (ETL)
Файл загружается через скрипт `load_data.py`.
- Используется библиотека **`ijson`** для потокового парсинга JSON, что позволяет обрабатывать файлы любого размера без переполнения RAM.
- Данные вставляются в БД пачками (batch insert) через `executemany` для скорости.

### 3. Text-to-SQL (LLM)
Основная логика преобразования текста в запрос находится в `llm_service.py`.

**Подход:**
1. Бот получает текст на русском языке.
2. Формируется запрос к LLM, содержащий:
   - Схему базы данных (DDL описание).
   - Системные инструкции (System Prompt).
   - Вопрос пользователя.
3. LLM возвращает чистый SQL-запрос.
4. Бот выполняет запрос и возвращает результат (число).

## 🧠 Промпт-инжиниринг и Схема данных

Для корректной работы модели используется детальный System Prompt.

**Как описана схема данных:**
Модели явно указано семантическое различие таблиц:
- Таблица `videos` — для статических вопросов ("сколько всего видео?", "сколько видео набрало > X лайков?").
- Таблица `video_snapshots` — для динамических вопросов ("на сколько выросли просмотры за дату X?", "прирост").

**Ключевые инструкции промпта:**
1. **Обработка дат:** Модели дано жесткое указание конвертировать русские даты (напр. "28 ноября 2025") в SQL формат `TIMESTAMP` (диапазон от `00:00:00` до `23:59:59`).
2. **Расчет прироста:** Если вопрос содержит слова "выросли", "прирост", "новые" — модель обязана использовать агрегацию `SUM(delta_...)` по таблице снепшотов, а не брать абсолютные значения.
3. **Безопасность:** Выполнение только `SELECT` запросов.
4. **Формат:** Строгий возврат только SQL кода без markdown-оберток.

## 🛠 Локальный запуск (без Docker)

Если вы хотите запустить проект без Docker:
1. Поднимите PostgreSQL локально.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Измените `.env`: `POSTGRES_HOST=localhost`.
4. Запустите миграции (выполните содержимое `init.sql`).
5. Запустите загрузчик: `python load_data.py`.
6. Запустите бота: `python main.py`.
